## 1. 产品概述

### 1.1 产品定位
Digital Twin 是一款面向个人用户的数字分身创建与管理平台，通过AI技术帮助用户创建个性化的数字分身，实现：
- **记忆留存**：永久保存个人故事、价值观、语言风格
- **情感连接**：让亲友在需要时仍能与"你"对话
- **时间穿越**：为未来的自己或他人留下信息

### 1.2 目标用户

| 用户群体 | 需求场景 | 核心痛点 |
|----------|----------|----------|
| 普通用户 | 为家人创建纪念分身 | 担心亲人离世后回忆消散 |
| 老年人 | 记录人生故事给孙辈 | 不会使用复杂工具，希望简单录制 |
| 内容创作者 | 创建AI助手与粉丝互动 | 无法24小时回复所有私信 |
| 远程工作者 | 让团队了解自己的工作风格 | 新同事难以快速理解自己 |
| 特殊职业者 | 医生/教师留下专业知识分身 | 经验传承困难 |

### 1.3 竞品分析

| 竞品 | 优点 | 缺点 | 我们的差异化 |
|------|------|------|-------------|
| HereAfter AI | 专注生前故事，产品成熟 | 价格高($99/年起)，仅英文 | 中文优化，价格亲民 |
| Replika | 情感陪伴强，用户粘性高 | 偏向虚拟恋人，不够正式 | 专注真实人格复刻 |
| StoryFile | 视频交互真实感强 | 需预录大量视频，成本高 | 纯AI生成，零拍摄成本 |
| 淘宝AI复活 | 价格低廉(20元起) | 质量参差不齐，隐私风险 | 质量保证，隐私可控 |
| 讯飞虚拟人 | 技术领先，形象逼真 | 企业导向，个人门槛高 | 个人友好，低门槛 |

### 1.4 核心价值主张

> **"让记忆超越生命，让对话跨越时空"**

- **简单**：30分钟完成创建，无需技术背景
- **真实**：多维度性格建模，还原真实人格
- **私密**：数据完全由用户控制，支持本地部署
- **持久**：一次创建，永久保存，多代传承

### 2.2 核心功能详解

#### 2.2.1 性格测试模块

**需求描述**
通过科学的心理学测评，快速构建用户性格画像。

**功能规格**

| 属性 | 规格 |
|------|------|
| 题目数量 | 25-30题 |
| 完成时间 | 5-8分钟 |
| 测评模型 | Big Five + 自定义维度 |
| 结果精度 | 95%以上用户认为"符合自己" |

**题目类型**
1. **选择题** (60%): "在社交场合，你通常..." A.主动交谈 B.等待别人来找你
2. **情景题** (25%): "朋友向你倾诉烦恼，你会..."
3. **排序题** (10%): 将以下价值观按重要性排序
4. **开放题** (5%): 用三个词描述自己

**测评维度**
```yaml
big_five:
  openness: 开放性          # 0-100
  conscientiousness: 尽责性  # 0-100
  extraversion: 外向性      # 0-100
  agreeableness: 宜人性     # 0-100
  neuroticism: 神经质       # 0-100

custom_dimensions:
  humor_style: [讽刺, 自嘲, 正经, 活泼]
  formality: 正式程度        # 0-100
  verbosity: 话痨程度        # 0-100
  emotional_expression: 情感表达 # 0-100
  conflict_style: [回避, 合作, 竞争, 妥协]
```

**验收标准**
- [ ] 支持中途保存，可继续答题
- [ ] 每道题可返回修改
- [ ] 结果页面展示性格雷达图
- [ ] 提供详细解读报告
- [ ] 支持"这不像我"反馈机制

#### 2.2.2 信息录入模块

**需求描述**
让用户补充个性化信息，提升分身真实度。

**信息分类**

| 类别 | 字段示例 | 必填 | 用途 |
|------|----------|------|------|
| 基础信息 | 姓名、昵称、出生日期 | 是 | 身份识别 |
| 人生经历 | 成长地、教育背景、职业 | 是 | 背景知识 |
| 重要事件 | 人生转折点、成就、遗憾 | 否 | 记忆素材 |
| 人际关系 | 家人、好友、重要他人 | 否 | 社交语境 |
| 兴趣爱好 | 喜欢的书/电影/音乐/食物 | 否 | 话题扩展 |
| 语言风格 | 口头禅、常用语气词、表情 | 否 | 语气还原 |
| 价值观 | 人生信条、重要原则 | 否 | 决策依据 |
| 禁忌话题 | 不想讨论的内容 | 否 | 安全边界 |

**输入方式**
1. **表单填写**: 结构化字段
2. **语音输入**: 讲述故事自动转文字
3. **文档导入**: 支持.txt/.md/.docx
4. **聊天记录导入**: 微信/QQ聊天记录解析

**智能辅助**
- 根据已有信息智能推荐补充问题
- 示例提示降低填写难度
- 实时预览效果

#### 2.2.3 分身生成模块

**需求描述**
基于测试和录入信息，AI生成可交互的数字分身。

**生成流程**
```
用户输入数据
    │
    ▼
数据验证与清洗
    │
    ▼
性格特征向量生成
    │
    ▼
记忆知识库构建
    │
    ├─────► 长期记忆(向量数据库)
    │
    ▼
语言风格模型训练
    │
    ▼
系统Prompt生成
    │
    ▼
对话能力测试
    │
    ▼
分身创建完成
```

**生成内容**

| 输出项 | 描述 | 存储位置 |
|--------|------|----------|
| Persona Profile | JSON格式的完整人格描述 | PostgreSQL |
| Embedding Vector | 768维向量表示 | pgvector |
| System Prompt | LLM系统提示词 | PostgreSQL |
| Memory Bank | 结构化记忆条目 | pgvector |
| Speech Pattern | 语言风格规则 | PostgreSQL |

**质量评估**
- 自动测试对话一致性
- 多样性检查（避免重复回答）
- 安全性检查（过滤有害内容）

#### 2.2.4 对话系统模块

**需求描述**
实现流畅、真实的对话体验。

**核心能力**

| 能力 | 描述 | 优先级 |
|------|------|--------|
| 上下文理解 | 记住对话历史，连贯交流 | P0 |
| 人格一致性 | 保持稳定的性格和语气 | P0 |
| 记忆检索 | 引用用户提供的个人信息 | P0 |
| 情感感知 | 识别访客情绪并适当回应 | P1 |
| 多轮推理 | 复杂问题的深度交流 | P1 |
| 知识扩展 | 在允许范围内使用通用知识 | P2 |

**对话流程**
```
访客发送消息
    │
    ▼
┌─────────────────────────────────────┐
│ 1. 意图识别                          │
│    - 分类：问候/提问/倾诉/闲聊        │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│ 2. 记忆检索 (RAG)                    │
│    - 查询相关记忆                    │
│    - 检索相似历史对话                 │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│ 3. Prompt构建                        │
│    - 系统提示 + 记忆 + 上下文         │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│ 4. LLM生成                           │
│    - 流式输出                        │
│    - 实时展示                        │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│ 5. 后处理                            │
│    - 内容安全过滤                    │
│    - 提取新记忆                      │
│    - 记录对话日志                     │
└──────────────┬──────────────────────┘
               │
               ▼
          返回响应
```

**WebSocket事件**
```typescript
// 客户端 -> 服务端
interface SendMessage {
  type: 'message';
  content: string;
  conversationId: string;
}

// 服务端 -> 客户端
interface StreamResponse {
  type: 'stream' | 'complete' | 'error';
  delta?: string;      // 流式片段
  fullContent?: string; // 完整内容
  error?: string;
}
```

#### 2.2.5 定时任务模块

**需求描述**
设置自动化消息，在特定时间触发。

**任务类型**

| 类型 | 触发条件 | 示例 |
|------|----------|------|
| 一次性 | 指定日期时间 | "2025年12月25日发送" |
| 循环 | 周期性执行 | 每周一早安 |
| 事件触发 | 生日/纪念日 | 每年生日祝福 |
| 条件触发 | 特定条件满足 | 访客首次访问欢迎 |

**任务配置**
```typescript
interface ScheduledTask {
  id: string;
  name: string;
  type: 'one_time' | 'recurring' | 'event' | 'conditional';
  trigger: {
    cron?: string;           // Cron表达式
    date?: Date;             // 一次性日期
    event?: 'birthday' | 'anniversary';
    condition?: string;      // 条件表达式
  };
  action: {
    type: 'send_message' | 'send_email' | 'webhook';
    target: string[];        // 接收者列表
    template: string;        // 消息模板
    variables: Record<string, string>; // 变量
  };
  status: 'active' | 'paused' | 'completed';
}
```

**消息模板变量**
- `{{receiver_name}}` - 接收者姓名
- `{{persona_name}}` - 分身名称
- `{{current_date}}` - 当前日期
- `{{days_since}}` - 距离某事件天数
- `{{custom_var}}` - 自定义变量

- 资源使用率

## 6. 风险与限制

### 6.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| LLM API不稳定 | 对话中断 | 多厂商 fallback |
| 响应延迟高 | 用户体验差 | 流式输出 + 缓存 |
| 成本超支 | 运营亏损 | 用量限制 + 成本告警 |
| 数据泄露 | 信任危机 | 加密 + 审计 + 合规 |

### 6.2 合规风险

- 遵守《个人信息保护法》
- 遵守《生成式AI服务管理暂行办法》
- 明确AI生成内容标识
- 建立内容审核机制

### 6.3 伦理考量

- 明确告知用户这是AI，非真人
- 禁止冒充他人创建分身（需身份验证）
- 建立投诉/举报机制
- 尊重逝者家属意愿
