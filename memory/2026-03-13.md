## 数字分身项目 - 极简原型开发

### 架构决策变更
用户最初想要一个复杂的分布式系统（PostgreSQL、Redis、Docker、微服务），但最终决定采用极简本地原型方案，核心诉求：
- 不需要服务器
- 本地能跑起来就行
- 原型最快验证效果
- 训练数据存本地文件都行

### 简化后的技术栈
- **后端**: Python + FastAPI（单文件 main.py）
- **数据**: SQLite + JSON 文件本地存储
- **前端**: React + Vite
- **AI**: Kimi API 直接调用
- **部署**: 本地两条命令启动（无 Docker）

### 已实现功能
1. 用户注册/登录（JWT 认证）
2. 创建数字分身
3. 问卷数据收集（10个基础问题）
4. 分身列表展示
5. 基础对话界面

### 待完成
- 接入 Kimi API 实现真实对话
- ChromaDB 向量检索集成
- 定时消息功能

### 数据存储位置
项目根目录 `data/` 文件夹：
- `users.json` - 用户列表
- `avatars/` - 分身配置和数据
- `chroma/` - 向量数据库（预留）

### 启动命令
```bash
# 后端
cd backend && python main.py

# 前端  
cd frontend && npm run dev
```
